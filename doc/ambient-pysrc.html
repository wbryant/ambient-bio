   2  AMBIENT v0.1: Active Modules for Bipartite Networks 
   3  Copyright 2012 William A. Bryant and John W. Pinney 
   4   
   5  This module undertakes simulated annealing on a metabolic model (arranged as a 
   6  bipartite network) to find connected components of the network with a coordinated 
   7  characteristic (for instance upregulation in a particular environment).  Edge 
   8  toggling is used to sample the space of possible active modules (based on the 
   9  work of Ideker et al).

  26  INSTALLATION 
  27  ============ 
  28  AMBIENT does not require installation, it can be run straight away.  It is 
  29  required that the AMBIENT directory is in the path when run from the command 
  30  line / that it is in the Python path when imported. 
  31   
  32  PREREQUISITES 
  33  ------------- 
  34  Due to compatibility issues in NetworkX it is recommended that Python 2.7 be 
  35  used, rather than Python 3 
  36   
  37  Required non-standard Python packages: 
  38   
  39  NumPy 
  40  libSBML 
  41  NetworkX

  43  Example use 
  44  =========== 
  45  Two sets of example data are available with AMBIENT; presented here is the set of commands 
  46  required to run the analysis of transcriptomic data for Mycobacterium 
  47  tuberculosis invading the Human Macrophage - see the README for reference details 
  48  (assuming you are in the C{examples/MTU} directory and C{ambient.py} is in your path): 
  49   
  50  C{######## 
  51  # Import ambient 
  52  import ambient as amb} 
  53   
  54  C{# Load metabolic network 
  55  G = amb.import_SBML_to_bipartite('inj661.xml')} 
  56   
  57  C{# Read reaction scores into the network 
  58  G = amb.read_rxn_scores(G, 'MTU_scores.txt')} 
  59   
  60  C{# Execute simulated annealing for positive and negative scores. 
  61  H_up, scores_up, cc_up = amb.ambient('MTU_example_up', G) 
  62  H_down, scores_down, cc_down = amb.ambient('MTU_example_down', G)} 
  63   
  64  C{# If a network export is required (for instance for visualisation) currently 
  65  # GraphML export is implemented} 
  66   
  67  C{# Map modules onto the metabolic network 
  68  G_class = amb.classify_nodes_ud(G, cc_up, cc_down, 'modules')} 
  69   
  70  C{# Export network to GraphML format.  GraphML cannot handle lists as attribute 
  71  # values, so the network attributes are all flattened.  The final output 
  72  # network is G_output 
  73  G_output = amb.gml_export(G_class, 'MTU_example.graphml') 
  74  ########} 
  75   
  76  N.B. the default value for C{N} (the number of iterations) is 10000, which may 
  77  take a few minutes to run.  This value is for testing, a more realistic value 
  78  for networks of this size would be ~2,500,000, but would depend on the underlying 
  79  structure of the network and the distribution of metabolite/reaction scores.

  81  MAIN FUNCTION DESCRIPTION 
  82  ========================= 
  83   
  84  C{H, scores, cc = ambient(expt_name, Q, N = 10000, M = 20, dir = 1, adaptive_interval = 3000, score_change_ratio = 0.2, intervals_cutoff = 6, H_edges = -1)} 
  85   
  86  Inputs 
  87  ------ 
  88  Required: 
  89   
  90  C{expt_name} - string - a name for output files to store the annealing results (no extension required) 
  91  C{Q} - networkx DiGraph - a bipartite metabolic network as imported by 'import_SBML_to_bipartite' 
  92   
  93  Optional: 
  94   
  95  C{N} - int - number of iterations. 
  96  C{M} - int - number of module scores to track in simulated annealing.  If M is set to -1, the simulated annealing is done using logarithmic scoring without limiting the number of modules. 
  97  C{dir} - int - if dir = -1, the negative of all reaction scores is taken before simulated annealing proceeds. 
  98  C{adaptive_interval} - number of steps to take before assessing how the score has changed for adaptive schedule. 
  99  C{score_change_ratio} - percentage rate of change per thousand steps below which temperature is dropped. 
 100  C{intervals_cutoff} - number of times to go through adaptive_interval number of steps before automatically dropping temperature. 
 101  C{H_edges} - list of tuples of 2 ints - a list of edges that if specified is used as the seed set of edges in the simulated annealing process. 
 102   
 103  Outputs 
 104  ------- 
 105  Output variables: 
 106   
 107  C{H} - networkx Graph consisting of the subnetwork induced by the final selected edge set. 
 108  C{scores} - list of floats consisting of the scores of the top 20 modules in the final subnetwork (in order high to low). 
 109  C{cc_out} - an ordered list of tuples where each entry contains the nodes in the 20 top scoring modules (in order high to low scoring). 
 110   
 111  Output files: 
 112   
 113  C{expt_name.log} which contains details of the edge toggling events that affected the top scoring module, which gives an indication of how the algorithm progressed. 
 114  C{expt_name.dat} which is a shelf file created by the shelve.open() command in the shelf module, which contains the three main outputs and the network used for the simulated annealing.
